# Free Pascal GO32v2 Setup

OS: go32v2
CPU: i386

## Requirements

- **fpcupdeluxe** to handle all the bootstrapping (I use v2.4.0g, but newer versions should work too)
- CWSDPMI or another compatible DPMI (DOS Protected Mode Interface) host

## Compiler Setup

**FPC setup**

1. Open **fpcupdeluxe**
2. Find a location that is easy to find, for example:
   `E:\fpc-go32v2-i386`
3. Choose either **stable** or **3.2.2**
   I chose this version because it's proven working
4. Click install **Only FPC**
5. Wait for at least 10-15 minutes until the compile process is finished

**Cross-compiler setup**

1. Click **Cross**
2. Choose CPU: **i386**
3. Choose OS: **go32v2**
4. Then click **Install compiler**
5. The process takes more or less the same amount of time as FPC setup above

## Verifying the build

First of all, make sure that the GO32v2 target is actually used, and the compiler doesn't fall back to the default Windows compiler

### Example minimum programme

Create a new unit, say `HELLO.PAS`

Copy & paste this in it:

```pascal
program HelloGO32;

uses CRT, DOS;

begin
  TextColor(yellow);
  writeln('Hello from GO32v2!');
end.
```

Then try compiling it with this command:

```
E:\fpc-go32v2-i386\fpc\bin\x86_64-win64\fpc.exe -Tgo32v2 -Pi386 .\HELLO.PAS
```

Because the generated `fpc.cfg` currently lacks the required unit paths, it should show a line mentioning an error:

```
Fatal: Can't find unit system used by HelloGO32
Fatal: Compilation aborted
```

Because of this, the unit directories must be added manually to either:
- Lazarus unit paths (if you use Lazarus IDE), or
- Manually supplying `-Fu` compiler options, or
- the compiler configuration `fpc.cfg` itself

The unit paths are these:
- `E:\fpc-go32v2-i386\fpcsrc\rtl\units\go32v2`
  - Contains the `DOS` unit (`dos.ppu`) and `System` unit (`system.ppu`)
- `E:\fpc-go32v2-i386\fpcsrc\packages\rtl-console\units\go32v2`
  - Contains the `CRT` unit (`crt.ppu`)
